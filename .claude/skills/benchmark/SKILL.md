# Design Benchmark

Перед реализацией любого крупного UI решения — сравни с лучшими командами мира.

## Эталоны и их принципы

### Linear
**Фирменный стиль:** Информационная плотность без перегруза. Каждый пиксель работает. Чёрный/белый как основа. Точечный фиолетовый акцент. Моноширинный шрифт для технических деталей.

**Что делает Linear:**
- Показывает продукт, а не обещания
- Плотные UI элементы без избыточного padding
- Анимации функциональные, не декоративные
- Текст конкретный: числа, факты, не абстракции

**Что НЕ делает Linear:**
- Иконки в секциях преимуществ
- Hero с иллюстрацией справа
- Testimonials в карточках
- Градиентные CTA кнопки
- "Why Linear?" секции

---

### Stripe
**Фирменный стиль:** Типографическая иерархия как главный инструмент. Минимум цвета. Код как дизайн-элемент. Синий акцент с точностью хирурга.

**Что делает Stripe:**
- Показывает реальный код/интерфейс
- Длинные headline без страха
- Нетривиальные layouts на больших экранах
- Цвет только для сигнала, не декора
- Responsive без жертв на мобиле

**Что НЕ делает Stripe:**
- Равномерные карточки 3x2
- Stock illustrations или generic 3D
- "Easy to use" / "Powerful" без доказательств
- Заполнять пространство ради заполнения

---

### Vercel
**Фирменный стиль:** Тёмная тема как default. Монохром + точечный акцент. Редакторская типографика. Простота на грани пустоты. Developer-first.

**Что делает Vercel:**
- Dark mode как идентичность, не тренд
- Минимальный текст, максимальный смысл
- CLI/terminal aesthetic
- Цифры без украшений

**Что НЕ делает Vercel:**
- Маркетинговые иллюстрации
- Большие CTA блоки с подложкой
- Feature grid с иконками
- Округлые happy-путь иллюстрации

---

### Anthropic
**Фирменный стиль:** Editorial sophistication. Длинные тексты — норма. Доверие через спокойствие. Беж/тёплые тона. Serif типографика.

**Что делает Anthropic:**
- Текст как контент, не как маркетинг
- Serif заголовки
- Нет суеты и анимаций ради анимаций
- Нет скидок на читателя

---

### Figma
**Фирменный стиль:** UI для профессионалов. Плотность + контроль. Показывает продукт честно. Яркий акцент строго функционально.

---

### OpenAI
**Фирменный стиль:** Minimal, clean. Sans-serif. Чёрный/белый. Серьёзность без пафоса. Chatbot как интерфейс, не виджет.

## Benchmark Checklist

Для каждого ключевого UI решения заполни:

```
BENCHMARK: [название компонента/секции]

Команда Linear реализовала бы так?     ДА / НЕТ — [почему]
Команда Stripe реализовала бы так?     ДА / НЕТ — [почему]
Команда Vercel реализовала бы так?     ДА / НЕТ — [почему]
Команда Anthropic реализовала бы так?  ДА / НЕТ — [почему]
Команда Figma реализовала бы так?      ДА / НЕТ — [почему]

SCORE: X/5

Если ≥3 НЕТ → БЛОК. Предложи альтернативу.
Если 2 НЕТ → ПРЕДУПРЕЖДЕНИЕ. Объясни риски.
Если ≤1 НЕТ → ПРИНЯТЬ.
```

## Быстрый тест "Would Stripe ship this?"

Три вопроса:
1. Показывает ли это реальный продукт или только обещания?
2. Есть ли здесь декоративные элементы без функции?
3. Прошло бы это дизайн-ревью в компании с высокой планкой?

Если 2+ "нет" → переработай.

## Отраслевые паттерны которые все три реджектят

- Hero с иллюстрацией справа
- 6 иконок в равной сетке
- "Trusted by 10,000+ companies"
- Карусель testimonials
- Одинаковые rounded-2xl карточки с shadow
- Gradient overlay на фото
- "Get started for free" как единственный CTA

## Применение

Запускай `/benchmark` перед реализацией:
- Hero секции
- Features/Benefits блока
- Pricing секции
- любого нового layout решения

Не запускай для: правок цвета, мелких типографических изменений, фикса багов.

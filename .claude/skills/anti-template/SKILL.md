# Anti-Template Enforcer

Задача: остановить шаблонное мышление до того как оно станет кодом.

## Запрещённые паттерны (встречаются на >30% AI/SaaS сайтов)

### Hero секция
- [ ] Заголовок Bold слева + иллюстрация/скриншот справа (2 колонки)
- [ ] Подзаголовок серым цветом под заголовком
- [ ] Две кнопки: Primary + Secondary (Get Started + Learn More)
- [ ] Gradient text в заголовке
- [ ] Floating UI элементы/скриншоты как иллюстрация

### Social Proof
- [ ] "Trusted by X,000+ companies" под hero
- [ ] Логотипы клиентов в строку grayscale
- [ ] Число пользователей как аргумент

### Features/Benefits
- [ ] 6 иконок в сетке 3x2 или 2x3
- [ ] Карточки с иконкой + Bold заголовок + серый описательный текст
- [ ] "Why [Product Name]?" как заголовок секции
- [ ] "How it works" в 3 шага с номерами в кружочках
- [ ] Numbered steps: 01, 02, 03

### Testimonials
- [ ] Карточки с аватаром + имя + должность + звёздочки
- [ ] Карусель отзывов
- [ ] Quote icon большой декоративный

### Pricing
- [ ] 3 тарифных плана: Starter/Pro/Enterprise
- [ ] Highlighted "Most Popular" на среднем
- [ ] Таблица с галочками ✓ и крестиками ✗

### CTA
- [ ] Большой блок в конце с градиентным фоном
- [ ] "Ready to get started?" как заголовок
- [ ] "No credit card required" под кнопкой

### Stats
- [ ] 3-4 числа в ряд: "1M+ Users", "99.9% Uptime", "4.9/5 Rating"
- [ ] Крупные числа с маленькой подписью

### Misc
- [ ] Rounded-2xl карточки с box-shadow
- [ ] Hover на карточках: поднятие + тень
- [ ] Glassmorphism фон за элементами без функции
- [ ] Gradient overlay на секциях

## Протокол проверки

### Быстрый тест (для любого нового компонента)

Отметь все найденные паттерны из списка выше.

```
ANTI-TEMPLATE SCAN: [компонент]

Найдено шаблонных паттернов: X/Y
Уровень шаблонности: LOW (0-2) / MEDIUM (3-5) / HIGH (6+)

Статус: ПРИНЯТЬ / ПРЕДУПРЕЖДЕНИЕ / БЛОК
```

**LOW** → принять, отметить что именно нестандартно.  
**MEDIUM** → предупреждение. Предложи чем заменить шаблонные элементы.  
**HIGH** → БЛОК. Полная переработка с constraint design.

## Constraint Design — выход из шаблона

Когда блок — задай вопросы:

**Вопрос 1 — Запрет карточек:**
> Как передать ту же информацию без карточек? (List / Table / Prose / Timeline / Grid разной высоты)

**Вопрос 2 — Запрет иконок:**
> Как обозначить фичи без иконок? (Числа / Буквы / Цвет / Типографика / Скриншоты)

**Вопрос 3 — Запрет кнопок-CTA:**
> Как призвать к действию иначе? (Inline link / Form / Terminal input / Interactive demo)

**Вопрос 4 — Запрет двух колонок:**
> Как организовать layout без стандартного split? (Stacked / Asymmetric / Magazine / Full-width)

**Вопрос 5 — Запрет стандартных секций:**
> Что если страница — это один длинный документ без секций?

## Допустимые исключения

Шаблонный паттерн допустим ТОЛЬКО если:
1. Явно переосмыслен (не copy-paste, а трансформация)
2. Обоснован функционально (пользователю так понятнее)
3. Единственный из шаблонных элементов на странице

Обоснование "так принято" = отказ.  
Обоснование "пользователи ожидают" = нужны данные, не интуиция.

## Примеры переосмысления

### Вместо "6 иконок в сетке"
- Одна большая интерактивная демо
- Список в стиле changelog: дата + описание
- Сравнительная таблица с реальными данными
- Скроллируемый список с hover-preview

### Вместо "Hero: текст + иллюстрация"
- Full-width terminal с анимированным вводом команды
- Видео продукта без звука, автоматическое
- Только типографика, никаких иллюстраций
- Интерактивный прототип прямо в hero

### Вместо "Trusted by логотипы"
- Конкретные case studies с числами
- Реальные цитаты с контекстом, не карточки
- GitHub stars / npm downloads как social proof
- Changelog с датами (доказательство активности)

# Innovation Review Gate

Запрещено реализовывать первое пришедшее в голову UI решение.

## Обязательный протокол перед любым крупным UI компонентом

### Шаг 1 — Определи что реализуешь

Назови конкретно: "Hero секция", "Карточки преимуществ", "Навигация", "CTA блок", и т.д.

### Шаг 2 — Найди 3 стандартных варианта (то, что делают все)

```
СТАНДАРТНЫЕ РЕШЕНИЯ (запрещены без обоснования):
1. [решение] → встречается на X% AI/SaaS сайтов → слабое потому что [причина]
2. [решение] → встречается на X% AI/SaaS сайтов → слабое потому что [причина]
3. [решение] → встречается на X% AI/SaaS сайтов → слабое потому что [причина]
```

### Шаг 3 — Найди 3 нестандартных варианта (выход за рамки шаблона)

Источники вдохновения: editorial design, luxury brands, gaming UI, fintech, physical product design, print media.

```
НЕСТАНДАРТНЫЕ РЕШЕНИЯ:
1. [решение] — вдохновение: [откуда] — почему работает: [причина]
2. [решение] — вдохновение: [откуда] — почему работает: [причина]
3. [решение] — вдохновение: [откуда] — почему работает: [причина]
```

### Шаг 4 — Constraint Design (обязательный вопрос)

Ответь письменно:

> Как бы выглядел этот компонент, если бы было ЗАПРЕЩЕНО использовать:
> карточки / иконки / градиенты / стандартные секции / типовые CTA / grid из равных блоков?

Этот ответ часто лучше чем первые 6 вариантов.

### Шаг 5 — Выбор

Выбери лучший вариант из нестандартных. Обоснуй:
- Почему это лучше стандарта?
- Как это выглядит у Linear/Stripe/Vercel?
- Что жертвуем и стоит ли того?

## Правила

**Запрещено использовать стандартное решение без явного обоснования.** Обоснование "так принято" не считается.

**Если решение встречается на большинстве AI-сайтов** → автоматически слабое → нужна альтернатива.

**Если решение выглядит как шаблонный лендинг** → нужна альтернатива.

**Минимум одна реализация должна выглядеть как продукт, а не маркетинговый сайт.**

## Примеры применения

### Hero секция

Стандарт: Заголовок + подзаголовок + CTA слева, иллюстрация/скриншот справа.

Нестандартно:
- Только типографика, никакой иллюстрации (как Vercel, Anthropic)
- Интерактивная демонстрация продукта прямо в hero (как Linear)
- Full-viewport видео/анимация с минимальным текстом
- Редакторский layout с большим номером или датой как доминирующий элемент

### Feature секция

Стандарт: 6 карточек с иконками в сетке 3x2.

Нестандартно:
- Одна большая фича + скриншот, остальные в списке
- Интерактивный переключатель с live preview
- Таблица сравнения (как продуктовая страница, не маркетинг)
- Scrollytelling — каждая фича открывается при скролле

# Политика безопасности

## Поддерживаемые версии

Поддерживается последняя минорная версия kit (см. [`.claude/kit-version`](.claude/kit-version)).

## Как сообщить об уязвимости

**Не открывай публичный Issue** для уязвимостей.

Используй приватный канал GitHub: **Security → Report a vulnerability** (Private vulnerability reporting) в этом репозитории. Опиши:

- затронутый файл/хук/skill,
- шаги воспроизведения,
- потенциальное влияние.

Ответ в течение разумного срока. После исправления — публичное раскрытие по согласованию.

## Область

Особое внимание:

- **Хуки** (`.claude/hooks/`, `.cursor/hooks/`) выполняются как код в среде пользователя.
- **`safety-guard`** блокирует деструктивные команды (`DROP TABLE/DATABASE`, `git push --force` на protected branches) — обход является уязвимостью.
- **Секреты**: токены хранятся в `*/secrets/github_token` и `*.env`, исключённых через [`.gitignore`](.gitignore). Случайный коммит секрета — сообщи по каналу выше.

## Хорошие практики при использовании

- Никогда не вставляй реальный PAT в файлы вне `secrets/`.
- Проверяй `.gitignore` перед первым коммитом в целевом проекте.
- Минимальный scope GitHub-токена: `contents:read`.
